# Vue TS Debug Workflow

## 默认约束
- 先稳定复现，再最小化复现，再修复。
- 调试结论必须有代码或日志证据。
- 默认中文回复，优先修根因。

## 执行步骤
1. 收集现象：报错堆栈、触发步骤、环境信息。
2. 划分层级：编译期、构建期、运行期、数据期。
3. 使用 DevTools 与日志定位首次异常点，而非末端报错点。
4. 先做最小修复，再补回归验证。
5. 输出时给出根因、修改点、验证路径、剩余风险。

## 质量检查
- 修复不引入新警告或类型回退。
- 能解释为何之前失败、为何现在通过。
- 关键路径至少完成一次手工回归。
