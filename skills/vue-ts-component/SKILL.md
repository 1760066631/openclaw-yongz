# Vue TS Component Standard

## 默认约束
- 使用 `<script setup lang="ts">`。
- 为 props、emits、slots 提供明确类型。
- 默认中文回复，优先最小改动与根因修复。

## 输入契约
- 组件用途：展示组件、表单组件、容器组件。
- 输入输出：props、事件、插槽、`v-model`。
- 交互状态：加载、空态、禁用、错误态。

## 执行流程
1. 定义组件契约
- 先定义 Props 和 Emits 类型。
- 对可选 props 设置合理默认值。
- 明确插槽入参与可选性。

2. 组织状态与逻辑
- 用 `computed` 表示派生状态。
- 用 `watch/watchEffect` 处理副作用并清理。
- 避免模板里堆复杂表达式。

3. 处理组件通信
- 子组件只通过 emits 通知变化。
- 父组件统一接收并处理业务副作用。
- `v-model` 按 `modelValue` / `update:modelValue` 规范实现。

4. 复用与抽象
- 重复逻辑提取为 composable。
- 重复结构提取为子组件。
- 控制抽象层级，避免过早通用化。

5. 组件验证
- 验证 props 边界值与默认值行为。
- 验证交互链路和事件参数。
- 验证 slot 缺省与自定义渲染。

## 质量门禁
- 不在子组件中硬编码父级业务规则。
- 不通过直接修改 props 驱动状态。
- 不让样式副作用污染其他组件。

## 交付模板
1. 组件契约：props/emits/slots 变化说明。
2. 交互说明：状态切换与关键事件。
3. 验证结果：关键场景已覆盖项。
4. 风险备注：未覆盖场景与建议。

## 反模式
- Props 类型与默认值冲突。
- Emits 名称语义不清或参数不稳定。
- 把业务流程直接写进通用组件。
