# Vue TS Pinia Standard

## 默认约束
- Store 按业务域拆分，不按页面临时拆分。
- 使用强类型 state/getters/actions。
- 默认中文回复，优先最小改动与根因修复。

## 输入契约
- 状态范围：全局共享、页面局部、临时 UI 态。
- 生命周期：是否需要刷新后保留。
- 并发行为：重复请求、竞态覆盖、取消策略。

## 执行流程
1. 划分状态边界
- 全局共享状态进入 store。
- 仅页面内部使用的状态留在组件。
- 避免把所有状态都上提到 Pinia。

2. 设计 store 结构
- State 只存源数据，不存可计算冗余。
- Getters 负责派生数据与读模型。
- Actions 负责异步流程与状态迁移。

3. 处理异步与并发
- 为异步 action 维护 loading/error/success。
- 对重复请求做去重、覆盖或取消策略。
- 失败路径恢复到可预测状态。

4. 持久化与同步
- 仅持久化必要字段，避免污染本地存储。
- 为版本升级设计迁移或兜底。
- 跨页面共享时保持更新路径单一。

5. 调试与验证
- 验证关键 action 的输入输出。
- 验证刷新后状态恢复。
- 验证失败场景不会残留脏状态。

## 质量门禁
- 不在组件中绕过 action 直接改关键状态。
- 不让 store 之间形成循环依赖。
- 不把临时 UI 状态过度持久化。

## 交付模板
1. Store 变更：state/getters/actions 调整点。
2. 异步策略：并发与错误处理规则。
3. 持久化策略：字段与生命周期说明。
4. 验证结果：核心状态流与边界场景。

## 反模式
- 把所有字段都放进 `persist`。
- 同一状态在多个 store 重复维护。
- action 内混杂过多页面 UI 逻辑。
