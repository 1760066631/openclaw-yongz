# Vue TS Vite Standard

## 默认约束
- 区分开发与生产环境，避免配置混用。
- 配置改动尽量局部化并可回滚。
- 默认中文回复，优先最小改动。

## 输入契约
- 问题类型：启动失败、热更新异常、构建失败、产物过大。
- 环境信息：Node 版本、包管理器、CI 与本地差异。
- 目标结果：修复错误、优化体积、增强调试能力。

## 执行流程
1. 分类定位
- 先判断在 dev server、build、preview 还是运行时暴露。
- 收集首个错误而非连锁错误。

2. 校验核心配置
- 校验 `resolve.alias` 与 tsconfig paths 一致。
- 校验 `server.proxy` 的路径重写和 `changeOrigin`。
- 校验 plugins 顺序与冲突。

3. 管理环境变量
- 使用 `VITE_` 前缀暴露到客户端。
- 将环境差异写入 `.env.*`，避免硬编码。
- 对关键变量做空值保护。

4. 优化构建
- 先做路由懒加载和按需引入。
- 再做 `manualChunks` 等分包策略。
- 保留必要 sourcemap 以便回归定位。

5. 验证结果
- 验证开发启动、构建、预览三链路。
- 对代理与环境变量做关键路径联调。
- 对体积优化给出前后对比。

## 质量门禁
- 不引入全局副作用插件替代局部问题修复。
- 不为临时问题永久关闭警告或检查。
- 不牺牲可调试性换取短期构建通过。

## 交付模板
1. 根因：问题定位到的配置点。
2. 改动：变更项与影响范围。
3. 验证：启动/构建/预览结果。
4. 观察：体积、速度或稳定性变化。

## 反模式
- 仅改 `vite.config.ts` 不同步 tsconfig paths。
- 在客户端读取非 `VITE_` 变量。
- 把代理问题误判为后端问题。
