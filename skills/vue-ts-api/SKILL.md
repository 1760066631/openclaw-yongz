# Vue TS API Standard

## 默认约束
- 页面层不直接拼 URL 或散落请求。
- 统一通过 API 模块暴露语义化方法。
- 默认中文回复，优先修根因。

## 输入契约
- 接口信息：方法、路径、参数、响应结构。
- 认证信息：token、刷新策略、权限约束。
- 错误规范：业务码、HTTP 码、提示文案。

## 执行流程
1. 定义类型
- 为请求参数定义 `RequestDTO`。
- 为响应定义 `ResponseDTO`，必要时映射为页面模型。
- 对枚举字段给出字面量联合类型。

2. 建立统一请求层
- 统一处理 baseURL、超时、headers、鉴权。
- 在拦截器里做错误归一化与日志补充。
- 对可重试接口加重试策略，仅限幂等请求。

3. 组织 API 模块
- 按业务域拆文件，不按页面临时拆。
- 命名保持 `get/list/create/update/remove` 一致。
- 导出函数返回稳定类型，避免透传底层库细节。

4. 页面联调
- 页面只调用 API 方法并消费结果。
- 将 loading、empty、error 状态收敛在页面状态机中。
- 提交场景覆盖成功、业务失败、网络失败。

5. 验证与回归
- 验证参数序列化与响应反序列化。
- 验证错误提示可理解且可定位。
- 验证接口变更不会横向污染其他页面。

## 质量门禁
- 不向上层泄露未经处理的底层错误对象。
- 不在页面里重复鉴权与错误映射逻辑。
- 不把后端字段名直接耦合到 UI 文案。

## 交付模板
1. 接口清单：新增或变更的 API 方法。
2. 类型清单：请求/响应/业务模型变化。
3. 错误处理：统一策略与用户提示。
4. 验证结果：联调路径与失败场景覆盖。

## 反模式
- 用 `any` 吞掉响应不一致问题。
- 每个页面单独写请求实例。
- 只处理 happy path，不处理失败路径。
