# Vue TS Router Standard

## 默认约束
- 统一使用 Vue Router 4。
- 路由元信息结构化管理（如 `meta.title`、`meta.auth`、`meta.roles`）。
- 默认中文回复，优先根因修复。

## 输入契约
- 路由范围：静态路由、动态路由、404/兜底路由。
- 权限模型：登录态、角色、菜单来源。
- 异常表现：白屏、重定向循环、参数丢失、权限越权。

## 执行流程
1. 梳理路由图
- 明确入口路由、布局路由、子路由关系。
- 标记需要鉴权与公共路由。

2. 规范路由定义
- 为路由 `name` 建立稳定命名规则。
- 组件加载优先懒加载。
- 对 `params/query` 定义读取与默认策略。

3. 规范导航守卫
- 先判断登录态，再判断角色权限。
- 重定向时保留目标地址用于回跳。
- 避免在多个守卫重复做同一权限判断。

4. 动态路由与菜单联动
- 将菜单与路由元信息统一建模。
- 动态注入路由前做去重与合法性检查。
- 刷新后确保动态路由能恢复。

5. 验证链路
- 验证首次访问、刷新、深链直达。
- 验证未授权访问与降级路径。
- 验证参数传递与回退行为。

## 质量门禁
- 不写硬编码权限分支散落各页面。
- 不让导航守卫产生无限跳转。
- 不让菜单配置与路由配置长期双维护。

## 交付模板
1. 路由变更：新增/调整/删除项。
2. 权限规则：鉴权与角色策略。
3. 验证路径：授权与未授权场景。
4. 风险说明：动态路由边界与后续建议。

## 反模式
- 在组件内重复做路由权限拦截。
- `name/path` 命名不稳定导致缓存失效。
- 忽略刷新后动态路由恢复逻辑。
